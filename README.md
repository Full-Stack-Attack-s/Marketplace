# Шаг 1: Добавление готового проекта

Открой GitHub Desktop. (https://desktop.github.com/download/)
В верхнем меню выбери File -> Add local repository...
Укажи путь к твоей папке с проектом и нажми Add repository.


# Шаг 2: Ежедневный рабочий цикл 

# 1. Синхронизация

Перед началом любой задачи убедись, что ты стоишь на актуальном фундаменте.

Проверь, что в левом верхнем углу (Current Branch) выбрана ветка main.

В правом верхнем углу нажми Fetch origin. Если на сервере есть чужие обновления, кнопка изменится на Pull origin — нажми её, чтобы скачать код.

# 2. Создание ветки

Никогда не пиши код в main.

 Нажми на вкладку Current Branch.

Нажми кнопку New Branch.

Введи название своей задачи (например, feature/users-login) и нажми Create branch. Программа автоматически переключит тебя на неё.

# 3. Написание кода и фиксация

Как только закончишь написание кода, разверни GitHub Desktop. В левой панели ты увидишь список всех файлов, которые ты изменил, удалил или создал.

В левом нижнем углу в поле Summary впиши суть изменений (например: «Добавлена форма авторизации») и желательно заполнять description.

Нажми синюю кнопку Commit to feature/... в самом низу.

# 4. Отправка на сервер
Твой коммит сохранен локально. Теперь его нужно закинуть на GitHub.

Нажми большую синюю кнопку Publish branch (или Push origin, если ветка уже публиковалась) в верхней панели или в главном окне.

# 5. Создание запроса на слияние (Pull Request)

Как только код улетит на сервер, прямо в главном окне GitHub Desktop появится кнопка Create Pull Request.

Нажми её. Программа сама откроет твой браузер на нужной странице GitHub.

На сайте останется только нажать зеленую кнопку Create pull request и назначить проверяющего.
